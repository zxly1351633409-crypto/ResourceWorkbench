# 资源入库工作台 · 使用指南

更新时间：2026-07-11　适用版本：0.3.1

## 1. 选择启动方式

- `启动-正式工作台.bat`：日常使用。若已经打包则运行 EXE，否则运行当前源码；设置保存在 `%LOCALAPPDATA%\ResourceWorkbench`。
- `启动-开发工作台.bat`：开发调试；数据与正式版隔离，保存在项目 `.runtime\development`。
- `启动-干净分发预览.bat`：每次创建全新的临时 profile，模拟公司电脑第一次打开，不删除其他运行数据。

第一次打开干净版时，资源库和待整理区域都应为空。进入左下角设置选择资源库根路径，保存后正式版下次启动会继续使用。

## 2. 顶部统一输入框

顶部不再有“添加待整理文件夹”“添加压缩包”按钮，也没有层级和搜索行。把来源粘贴到同一个输入框：

```text
D:\待整理\一批资源
D:\下载\resource.7z
D:\批次一 ; E:\批次二
https://shotdeck.com/...
```

- 多个来源用分号或换行分隔。
- 点击“分析”或按 Enter 开始。
- 输入框失焦不会开始分析，因此第一次点击不会再被误当作取消。
- 网址和本地路径可以放在同一批输入中。

## 3. 分析与取消

分析运行时：

- 卡片区域顶部显示“正在分析中，请稍后”的流光文字。
- 下方显示进度条和“取消分析”。
- 顶部“分析”按钮暂时禁用，取消使用独立按钮，不再一键两用。
- 目录扫描和压缩包命令不会弹黑色控制台窗口。

点击取消后，工作台会在当前安全检查点停止。已经生成的卡片仍可查看；正常取消不会弹“分析未完成/用户取消”的警告框。

## 4. 卡片边界与压缩包

默认策略是“外层资源优先”：

- 外层一个资源文件夹对应一张卡片。
- 压缩包作为该资源的一部分或预览来源，不读取内部目录来拆卡。
- `textures`、`maps`、`normal`、`roughness`、`basecolor`、格式/工程目录不会成为独立资源卡。
- 卡片右键对压缩包显示“分析压缩包外层（当前不解压）”；它只重新分析压缩包文件本身和外层元数据，不会读取、拆分或解压内部目录。
- 普通分析和卡片“重新分析”都不能自动解压。即使旧设置中残留自动解压字段，v0.3.1 也会强制关闭。
- “深度读取压缩包内部”以后可作为独立、默认关闭的高级流程；当前尚未开放。

如果资源已手动解压，直接分析解压后的外层目录，识别信息会更完整。

## 5. 网页资源

粘贴 `http://` 或 `https://` 链接后点分析，会生成网页虚拟卡片：

- 优先读取标题、描述、平台、作者和网页封面。
- ShotDeck、Superhive 等站点会先尝试无界面网页截图。
- 如果截图是 Cloudflare/安全验证/封锁页，会丢弃该截图并生成风格化封面，不使用 favicon 冒充内容图。
- 卡片“打开”会访问原链接；选择目标分类后可保存为网页资源文件夹。

## 6. 资源库与卡片墙

- 左侧“资源库”有独立容器。展开三角和文件夹按需加载，不预扫整棵网络盘目录。
- 目录枚举和“是否还有下级目录”探测都在后台运行；慢 NAS 会先显示“正在读取…”。同一层超过 400 个文件夹时显示“继续加载（剩余 N）”，读取失败可点击重试。
- 资源库标题右侧的“＋”会在当前选中目录中真实新建文件夹；树的右键菜单也提供“新建文件夹 / 刷新当前路径”。
- 软件监听资源库根、当前目录和已展开目录；Windows 文件夹中新增/删除内容后会自动同步。NAS 监听失效时，后台轻量轮询会兜底，不需要再进设置重复保存路径。
- 点击任意深度的叶目录都会索引直系文件。纯视频目录显示“视频”卡并从视频抽帧；PNG/PSD 等图片目录显示图片预览。
- 缩略图生成在限并发后台队列中进行，浏览 4K 图片或多个视频时界面仍可操作。
- 点击资源树只切换“资源库”快速浏览，不会删除“待整理”视图中已经分析的卡片。
- 顶部“待整理 / 资源库”按钮可在两个视图之间切换。
- 卡片单击选择，双击打开详情；悬停或右键可翻译、修改目标、移动、打开来源和编辑标签/备注。
- 翻译导致本地文件夹重命名后，左侧树、输入路径、卡片来源和资源索引会同步刷新。
- SpeedTree `.spm` / 二进制 `.srt` 会优先使用工程同目录的 `preview.png` 或 `<工程名>_preview.png`，不会把 AO/Normal/Gloss/Opacity 等贴图当模型封面。没有真实图时显示明确标注的非渲染占位封面；工作台不会自动操控 SpeedTree 图形界面。

## 7. 翻译与目标推荐

在设置中配置 DeepSeek：

- 默认调用 Flash 档；Pro 档需要手动选择。
- API Key 写入当前运行环境的 `workbench_data/secret.json`，环境变量优先。
- 翻译命名支持中英顺序和仅中文/仅英文。

目标分类选择器会综合：

- 卡片标题、父批次语境、普通文件样例、压缩包文件名、扩展名和判断原因；
- 当前资源库真实的二级至四级分类目录；
- 用户过去已经成功并校验通过的移动记录。

普通“分析”完成时就会写入一个首选深层分类和若干备选，不再只给出 `M 模型` 这类大类。卡片详情会说明每个候选来自标题语义、批次语境还是历史习惯；候选必须是资源库中已经存在的分类目录。

历史命中会显示为“习惯推荐 ×次数”。撤销或校验失败的移动不会继续训练推荐。

## 8. 移动、日志与撤销

- 每次移动前都会先显示计划预览，逐项列出“来源 → 实际目标”、文件数和容量；预演失败不会进入执行阶段。
- 正式入库是实际改动文件的位置：看完计划后还必须在第二个确认框完整输入大写 `MOVE`。关闭预览、取消输入或输入其他文字都不会移动任何文件。
- 测试移动仍显示相同计划预览，但只需要普通确认，不要求输入 `MOVE`。
- 移动后重新统计文件数和容量，写入 SQLite 移动日志。
- 操作记录窗口可查看和撤销仍可回滚的移动。
- 移动成功后，待整理卡片会从当前墙中移除。
- 不确定卡片建议先进入审阅队列，不要批量直接移动。

## 9. 主题与颜色

设置 → 界面：

- 主题、强调色、面板色、画布色、按钮色保留为快捷设置。
- “语义颜色”可分别设置窗口、侧栏、卡片、输入框、主要文字、弱提示文字、边框、图标、按钮悬停和按钮/卡片选中态。
- 自定义配色窗口按“表面 / 文字与线条 / 交互状态”分组，提供浅色/深色预设、单项恢复和实时小预览。
- 留空表示跟随当前主题；“全部跟随当前主题”不会修改资源数据。标准按钮与颜色选择器均显示中文。

## 10. 缓存与日志清理

设置 → 行为中可控制：

- 预览缓存最大 MB 与最长保留天数。
- 移动日志最大条数与最长保留天数。
- “立即清理工作台运行数据”。

检查和清理都在后台运行。除预览、报告和已撤销移动历史外，还会保守处理：完整且过期、没有活动标记的 staging 批次；过期资源索引；明确为已完成/已撤销/已上传的终态历史；以及 SQLite 空闲页。仍可撤销的移动、活动/失败/未知状态记录、人工标签和卡片备注永久保留。清理范围只有工作台运行数据，不包含待整理目录和资源库。

## 11. 当前限制

- 默认不自动解压；运输压缩包/资源内容压缩包的全自动判定仍需继续完善。
- DeepSeek 主要做文本翻译与候选分类，不替代人工多模态判断。
- 115 真正上传流程仍未作为默认入口开放。
- 部分登录站点无法取得真实页面截图，此时使用风格化封面并保留原链接。
- SpeedTree 默认只复用已有真实预览；生成新的真实渲染需要管理员另行配置经过许可审核的无界面生成器。
