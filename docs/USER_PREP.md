# 你需要准备什么

第一轮开发测试，你准备这些就够了：

1. 2 到 3 个测试批次路径  
   最好分别包含：
   - 纯模型资源。
   - 纯照片资源。
   - 混合资源，里面有教程/照片/模型/多层压缩包更好。

2. 每个测试批次的正确答案  
   不需要写很正式，只要告诉我：
   - 这些资源最终应该算几张卡片。
   - 每张卡片大概是什么类型。
   - 应该移动到 `Z:\整合——资源管理` 的哪个分类。
   - 哪些内部压缩包应该保留，哪些只是运输包。

3. DeepSeek API Key  
   现在还不用发。等做到翻译/命名建议阶段再给，并且最好放到本地 `.env`，不要直接写进代码。

4. 115 上传方式选择  
   后面要确认你希望走哪一种：
   - 官方开放平台 API：更稳定，推荐。
   - 115 客户端界面自动化：能模拟你的手动流程，但更脆弱。

5. Mcc 文件名翻译工具的设置  
   因为它是闭源软件，我需要你之后演示或截图这些内容：
   - API 设置页。
   - 翻译规则。
   - 是否递归翻译。
   - 重名处理方式。
   - 中英并存命名格式。

6. QRoundedFrame 授权记录  
   你已确认作者授权可自由使用。建议保留一张聊天截图或授权说明，后面打包软件时更安心。
